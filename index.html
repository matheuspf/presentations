Análise de complexidade

  • Saber se um algoritmo irá executar no tempo determinado
  • Saber como o tempo do algoritmo escala de acordo com sua entrada
Análise de complexidade

  • Como base, podemos executar 100 milhões de operações por segundo em um PC convencional
  • Esse número pode variar bastante dependendo do que está sendo executado, então é bom ficar atento
Linguagem de programação

  • Sempre use C++, Java ou Python
  • C é totalmente desnecessário em programação competitiva
  • Para problemas pequenos, é possível utilizar um subconjunto de C++ com a mesma performance
Linguagem de programação

  • POO
  • Classes e funções já feitas
  • Facilidade de abstração
Backtracking

  • Procurar de forma exaustiva todas as possíveis soluções dentro de um espaço
  • Pode ser recursiva ou iterativa (recursive é bem mais comum)
